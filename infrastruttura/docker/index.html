    Docker¶

    Echidna è già pronto per utilizzare docker tramite docker-compose

    Le impostazioni dei servizi utilizzati sono nel file /docker-compose.yml.

    Per avviare Echidna in docker eseguire questo comando :

    docker-compose up

    Verranno avviati i seguenti servizi :

    web¶

    Il servizio web contiene il server apache e l'interprete php. In avvio monta la cartella /app dentro il container al path /var/www/html.

    Il servizio web si appoggia sul dockerfile /docker/web/Dockerfile e contiene delle configurazioni dell'host nel file /docker/web/confs/000-default.conf

    Cambiare porta¶

    Normalmente il servizio web si avvia in ascolto sulla porta 8080, che viene mappata sulla porta 80 all'interno del container.

    Per modificare la porta di ascolto sul proprio sistema basta cambiarla dentro il file docker-compose.yml ed eseguire una nuova build.

    ports:
       - "8080:80" # in ascolto sulla porta 8080

    per utilizzare invece la porta 80

    ports:
       - "80:80"

    db¶

    Il servizio db utilizza l'immagine mariadb per fornire il server mysql.

    Viene inoltre creato un volume persistente mariadbdata che conterrà tutti i dati di mysql anche dopo un riavvio.
