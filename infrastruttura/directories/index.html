    Le principali directories¶

    /app¶

    Contiene tutta la struttura della tua applicazione Echidna.

    applications¶

    Echidna si basa su applicazioni che hanno il compito di gestire autonomamente entità collegate al db, sia lato backend che lato frontned.

    Ogni cosa è gestita tramite la propria applicazione, anche gli aspetti più essenziali di Echidna come utenti, login e impostazioni.

    In questa cartella risiedono le applicazioni core disponibili e condivise con tutti i template frontend.

    backend¶

    Contiente il template che genera il backoffice. Al suo interno sono presenti anche i template per generare i fields che popolano i form di modifica.

    cache¶

    Cartella per la cache del sistema. Vengono salvate in questa cartella le cache relative alle query, impostazioni e pagine complete.

    La gestione di quale meccanismo di cache attivare è delegato all'applicazione Settings e disponibile in un pannello apposito nel backend.

    config¶

    In questa cartella risiedono i file di configurazione di Echidna e delle applicazioni a livello globale. Parleremo più avanti della configurazione di Echidna ma i file presenti in questa cartella agiscono a livello globale e possono essere però sovrascritti dal singolo template frontend.

    I file di configurazione minimi sono: - applications -> contiene le applicazioni da caricare - db -> contiene i dati di accesso al db - environment -> contiene le configurazioni dell'ambiente di sviluppo - response -> dichiara quali template utilizzare per json, backend e frontend

    core¶

    Contiene tutti i file core di Echidna

    frontend¶

    Contiene delle subdirectories che rappresentano i template frontend disponibili.

    install¶

    Contiene la procedura guidata di installazione del sistema. Dopo l'installazione iniziale è caldamente consigliato rimuovere questa directory.

    media¶

    Contiene tutte le immagini caricare da backend tramite l'applicazione Media e disponibili direttamente da web tramite url generate in questo modo /media/{perscorso interno}

    resize¶

    Cartella generata automaticamente dall'applicazione Assets ( TODO : spostare gestione in applicazione media) che contiene le versioni ridimensionate dei file media.

    I file vengono generati su richiesta tramite una url costruita con questo pattern :

    /resize/{impostazioni}/media/{percorso immagine}

    Una volta generata, l'immagine verrà servita direttamente da apache senza passare dall'applicazione Assets

    runtime¶

    Cartella utilizzata dal servizio RuntimeService dove vengono salvate informazioni utili all'esecuzione dell'applicazione.

    vendor¶

    Contiene le dipendenza composer di Echidna

    /docker¶

    Contiene i dockerfile per i servizi docker web e mariadb
