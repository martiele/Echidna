    Creare un nuovo template frontend¶

    Preparazione¶

    Per prima cosa conviene recarsi nel backend all'indirizzo /backend/settings/cache e controllare che la modalità sviluppatore sia attiva e tutti i sistemi di cache siano disattivati.

    Struttura directory¶

    Per creare un nuovo template frontend occorre agire nella cartella / app/frontend. In questo tutorial creeremo un nuovo template frontend denominato "primo".

    Prima di tutto creare una nuova cartella con il nome del template con all'interno la seguente struttura

    /app/frontend/
        | primo
            | assets
                | css
                    | style.less
            | template
                | pages
                    | home.twig
                    | default.twig
                notfound.twig
            PrimoTemplate.php

    File PrimoTemplate.php¶

    Questa è la classe base del template, deve estendere uno dei template engines disponibili. Questo il contenuto per il nostro esempio

    <?php 
namespace frontend\primo;
use core\template\TwigTemplate;

class PrimoTemplate extends TwigTemplate{
    function getTemplatesDirectory()
    {
        return __DIR__."/template";
    }

    static function getBaseDirectory()
    {
        return __DIR__;
    }
}

    Nota bene

    Il nome della classe deve essere uguale al nome del file ed il namespace deve rispecchiare la struttura directory in cui risiede

    Utilizzare il nuovo template¶

    Per utilizzare il nuovo template dobbiamo dichiararlo dentro al file di configurazione response, situato in /app/config/response.php

    Nel nostro esempio questo file avrà questo contenuto:

    <?php
use frontend\primo\PrimoTemplate;

return [
    "templateEngines"   =>  [
        "frontendTemplate"  =>  PrimoTemplate::class,
        "backendTemplate"   =>  \backend\BackendTemplate::class,
        "jsonTemplate"     =>  \core\template\JsonTemplate::class
    ]
];

    Controllare che il template funzioni¶

    Andando adesso sulla root del sito dovremmo vedere semplicemente una pagina bianca, questo perchè nei nostri file template non abbiamo inserito nessun contenuto.

    Modifichiamo il contenuto del file /app/frontend/primo/template/pages/home.twig scrivendoci una semplice stringa "Hello world!"

    All'indirizzo del sito adesso dovremmo vedere una pagina bianca con la scritta Hello World

    Template Twig¶

    Ogni template viene richiamato direttamente dall'applicazione che gestisce la route. Possiamo quindi scrivere template diversi per ogni pagina, creare pagine statiche o dinamiche oppure utilizzare la sintassi twig per estendere o includere altri template.

    Ereditarietà in twig¶

    Modifichiamo la struttura della cartella template come mostrato qui di seguito, aggiungendo un file template base

        | assets
    | template
        | pages
        | notfound.twig
        | base.twig
    PrimoTemplate.php

    In questo template abbiamo definito la struttura base dell'head e dichiarato un blocco "content" che potrà essere utilizzato nelle pagine per scrivere il contenuto.

    Il template riceve sempre alcune variabili in più a quelle gestite dalle singole applicazioni ( per questo vedere la sezione relative al servizio ResponseService ) che possono essere utilizzate. Quelle più importanti sono: - meta -> oggetto contenente i meta contenuti per la pagina - current_url -> url corrente - router_service -> istanza del servizio RouterService

    Il link stylesheet "/assets/css/style.css" viene mappato per sul nostro file "/assets/css/style.less".

    Estendere il file base¶

    Possiamo adesso cambiare il contenuto del template /pages/home.twig per fargli estendere il nostro template base e aggiungere del contenuto.

    {% extends "base.twig" %}

{% block content %} 
    Io sono il contenuto
{% endblock %}

    Includere un template¶

<div class="container">

    <div>
    {% block content %}
    {% endblock %}
    </div>
</div>

</html>

    Css¶

    I file css del template risiedono nella cartella /assets/css.

    Echidna ha integrato il preprocessore less per aiutare nello sviluppo css. E' quindi possibile lavorare direttamente con i file css oppure con file less che verranno convertiti da Echidna stesso.

    style.less¶

    Nel nostro template base abbiamo inserito un link al file "/assets/css/style.css" che al momento non esiste su disco. Questo file viene generato partendo dal file style.less.

    Prime regole css¶

    Proviamo ad impostare alcune regole css dentro il file style.less

    body{
    color: #fff;
    background: #000;
}

    Includere file less¶

    E' buona norma non includere tante risorse esterne nei siti web per velocizzarne il caricamento. Possiamo utilizzare la capacità di less di includere file esterni e compilarli in un unico file. style.less

    body{
    color: #fff;
    background: #000;
}
@import "contenuto.less";

    contenuto.less

    .contenuto{
    background: #f00;
    color: #000;
}

    Modifichiamo il file base.twig per vedere il risultato applicato alla nuova classe contenuto

    {% include "base/head.twig" %}
<div class="container">
    Io sono il template
    <div class="contenuto">
    {% block content %}
    {% endblock %}
    </div>
</div>
</body>
</html>
